# 宠物日记小程序（Pet Diary）

一个面向宠物主人的微信小程序方案，用于记录宠物日常、管理宠物档案，并提供关键事项提醒（生日、疫苗、清洁等）。

## 目标用户

- 个人宠物主人（猫狗为主，也支持其他宠物）
- 家庭多成员共同照护宠物
- 需要周期性提醒（疫苗、驱虫、洗护、体检）的用户

## 核心功能规划

### 1. 宠物档案

- 宠物基础信息：昵称、头像、品种、生日、性别、绝育状态
- 健康信息：过敏史、慢性病、常用药、医院档案
- 生命周期记录：领养日、绝育日、首次疫苗等

### 2. 日常发布（动态）

- 图文发布：记录宠物日常、训练进展、饮食变化
- 标签系统：如「饮食」「训练」「生病」「洗护」
- 时间线浏览：按日期查看成长轨迹

### 3. 提醒系统

- 生日提醒（按年）
- 疫苗提醒（支持下一针日期）
- 清理提醒（耳道、牙齿、洗澡、美容）
- 驱虫提醒（体内/体外）
- 自定义提醒（例如复查、补货）

### 4. 数据统计

- 近 30 天记录次数
- 提醒完成率
- 健康事件统计（就诊、用药、异常）

## 信息架构（建议）

- 首页：今日提醒 + 最近动态
- 记录页：发布动态、历史时间线
- 提醒页：全部计划、待办、已完成
- 宠物页：宠物档案、健康信息、设置

## MVP（建议 4 周）

### 第 1 周：基础架构

- 小程序项目初始化
- 用户登录与基础数据模型
- 单宠物档案创建/编辑

### 第 2 周：日常记录

- 图文动态发布
- 时间线展示
- 标签筛选

### 第 3 周：提醒功能

- 提醒创建/编辑/删除
- 到期提醒列表
- 完成打卡

### 第 4 周：体验优化

- 首页聚合信息
- 基础统计页面
- UI 优化与真机测试

## 技术实现建议

### 前端（小程序）

- 原生小程序 + TypeScript（推荐）
- 状态管理：按页面拆分，避免全局耦合
- 组件化：宠物卡片、提醒卡片、动态卡片

### 后端（可选）

- 微信云开发（快速上线）或独立后端（Node.js + MySQL）
- 对象存储：用于宠物图片与动态图片

### 数据模型（简化）

- `users`：用户信息
- `pets`：宠物档案
- `posts`：日常动态
- `reminders`：提醒任务
- `reminder_logs`：提醒完成记录

## 后续可扩展方向

- 多宠物家庭管理
- 成员协作（家人共同维护）
- 宠物医院对接（疫苗记录导入）
- AI 识别（便便状态、体态变化记录）
